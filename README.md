# Smart Telegram Crawler: Topic-Targeted Discovery

This repository contains a topic-targeted Telegram crawler designed to navigate the Telegram ecosystem, discover relevant channels, and filter out off-topic content. It was developed to assist OSINT researchers, Cyber Threat Intelligence analysts, and Data Scientists in efficiently collecting data from the platform.

The project was developed as part of a Master's Degree thesis in Cybersecurity at Politecnico di Torino to address the challenge of scraping large amounts of Telegram data while minimizing the computational overhead caused by irrelevant channels.

## Overview

Standard snowball sampling on Telegram often results in the collection of numerous groups irrelevant to the research topic. This crawler mitigates the issue by utilizing a threshold-based Breadth-First Search (BFS) traversal strategy, combined with machine learning and Large Language Models (LLMs).

The system tracks the flow of forwarded messages between channels, treating this as a parent-child relationship. The crawler proceeds to a child channel only if the parent channel contains a sufficient percentage of messages related to the target topic.

## Key Features

*   **Dual-Track Text Preprocessing**: Aggressively cleans text (removing URLs, stopwords, punctuation) for statistical topic detection, while preserving natural sentence structure for LLM context reading.
*   **Optimized Topic Modeling (LDA + TPE)**: Utilizes Latent Dirichlet Allocation (LDA) for rapid topic discovery, chosen for its processing speed compared to transformer-based models. Hyperparameters are automatically optimized using the Tree-Structured Parzen Estimator (TPE) via Optuna.
*   **LLM-based Classification**: Integrates an LLM to automatically classify the topics generated by LDA as relevant or irrelevant based on a customizable prompt.
*   **Resource Efficiency**: By halting exploration on network branches that fall below a predefined threshold (e.g., 40%), the crawler significantly reduces the required computing resources and execution time.

## Repository Structure

The workspace is organized as follows:

*   `environment/`: Contains conda and pip environment files for dependency management across different operating systems.
*   `scripts/`: Contains the core execution files.
    *   `0_...` to `7_...`: Scripts dedicated to preprocessing, strategy setup, hyperparameter optimization for LDA/BERTopic, and model extraction.
    *   `automatic_crawling/`: Contains the fully automated pipeline. `master_orchestrator.py` manages the end-to-end execution.
*   `result_visualization/`: Jupyter notebooks used for exploring results, generating heatmaps, analyzing graph structures, and comparing topic modeling performances.

## Architecture Pipeline

1.  **Seed Initialization**: The process begins with a curated set of initial seed channels.
2.  **Preprocessing**: Applies the dual-track text cleaning pipeline.
3.  **LDA Training & Inference**: Automatically identifies semantic topics within the current level's messages.
4.  **Topic Classification**: An LLM reads the top keywords and representative messages of a topic to determine its alignment with the target domain.
5.  **Group Classification**: The crawler computes the ratio of related messages inside the channel. The channel is marked as relevant if this ratio exceeds a predefined threshold.
6.  **Next-level Creation**: Extracts all forwarded channels from the relevant parents to create the seed list for the subsequent depth level.

## Installation & Setup

1. **Clone the repository:**
   ```bash
   git clone https://github.com/iam-MA/Smart-telegram-crawler.git
   cd Smart-telegram-crawler
   ```

2. **Create the environment:**
   Depending on your OS, use the provided Conda environment files located in the `environment/` folder.
   ```bash
   conda env create -f environment/linux_environment.yml
   # or
   conda env create -f environment/local_environment.yml
   
   conda activate <env_name>
   ```

## Usage

To run the automated crawling process, navigate to the execution directory and use the provided bash script or orchestrator token. Ensure necessary environment variables or LLM API keys are configured before proceeding.

```bash
cd scripts/automatic_crawling/
bash run_all_experiments.sh
# or run the orchestrator directly
python master_orchestrator.py
```

LLM prompts can be customized (e.g., modifying `prompt_gaming.txt` or `6_promp_personalized.txt`) to adapt the crawler to different domain requirements.

## Evaluation & Results

The system was evaluated on two large-scale datasets:
*   **US Dataset**: Focused on the 2024 US Presidential Election (over 500,000 political messages). Setting a 40% threshold effectively retained approximately 99.8% of target political communities while discarding off-topic groups.
*   **TGDataset**: A highly heterogeneous dataset containing over 120,000 channels and 500 million messages. By targeting the broader "Gaming" category, the crawler successfully identified narrow sub-topics such as "Videogame Modding".
